Variable - is a temporary storage location (in the computers RAM..) for data.  The text book uses the analogy of the Excel spreadsheet cells as a "storage location" just like a variable.  It is important to also understand data types when discussing variables.

Data types - are the type, kind, category, (...there are so many synonyms that could be used here...) of value that is stored in a variable or object property.  For example, if you wanted to know your monthly salary but all you have is the total amount you earned each week for four weeks, you would need to do "addition."

  • Week One: $125.52
  • Week Two: $125.81
  • Week Three: $100.74
  • Week Four: $125.30

You could add these values because they are numeric 'types' of data.  What you could not add is the One, Two, Three, and Four, because they are character (often known as 'string') 'types' of data.  Depending on the programming language (...or database management system) there are many data types.  Common data types include: numeric (integer, fixed-point decimal, floating-point decimal, etc.), string (character... including symbols that represent numbers), date (including time), boolean, binary (object), etc.

Constant - is the conservative sibling to a variable.  Where the variable is more liberal in its behavior of changing when necessary the conservative is more old fashioned and remains steady in its value.  (While the program/application is actively processing... the value of a variable may change, for a variety of reasons... thus the name variable.  A constant, on the other hand, acts like its name and does not change its value at ANY TIME the program is actively processing. 

A good example of a variable would be the price of an item at a hardware store.  If you purchase 3 different items, the clerk will enter the price of each item separately in the cash register as item 1, item 2, item 3 (these are variables as you'll see in a moment.)  A good example of a constant would be the state sales tax, which is the same value (...a numeric percentage) for each item, regardless of the price.  The clerk uses the values of the 3 variables and the 1 constant and determines a total cost (...incidently, another variable!)

A second customer immediately follows you through the checkout line with 3 entirely different items.  The clerk will enter the price of these each of these items into the cash register as item 1, item 2, and item 3... but the sales tax will still be the same value used for your 3 items.  (I hope this helps in your understanding of both variables and constants.)

Option Explicit - is a Visual Basic command that acts like a light switch in that if the command is issued (at the beginning of a program... in the General Declarations module of the code...) it reguires that most... there are very few exceptions that will be discussed in a later chapter... variables must be 'declared' before they can be used in procedures.  This is a very good practice because if it is not 'on' then the programmer can 'make it up as they type' and a variable named item1 in one line of code might be referenced as itme1 later in the code and the computer would not realize the programmer made a typing error.  The program would run, possibly without failure.  The result would be, likely, invalid results from the program.

Module - is a unit of programming code (declarations and procedures) that 'are related' usually by an object, such as a worksheet in Excel, or for a specific purpose, such as converting units of measure.  In VBA for Excel, modules are automatially created for each workbook and worksheet... whether we use them or not.  Because these modules pertain to specific 'objects' they are referred as object modules.  On the other hand, an example of a standard module would be the General Declarations module mentioned in the explanation of the Option Explicit command.

Scope - as it pertains to variables and constants, is the (program processing...) time when a particular variable is visible (accessible) to the program.  For example, if you declare/define a variable in the Worksheet1 module and then reference it in the Worksheet2 module, it may not be visible/accessible... depending on the how they were declared: Dim, Private, Public... each will be discussed in a moment.

Here's an example:  The United States is "a nation of laws."  (Well, we here that often from our politicians but it's just a 'catch phrase' if those laws aren't enforced... let's assume, for this example... they are.)

    • If the law is declared at the national level... each state, county, and municipality can see/access... and abides by the law.
    • If the law is declared at the state level... each county and municipality can see/access... and abides by the law... but the nation does not.
    • If the law is declared at the municipal level... only that municipality can see/access the law... so it only pertains to them... the state and nation does not have to abide by the law.
      • Sometimes... a law declared at a lower level, might be found (...by our judicial system) to be applicable to other municipalities, states, or the nation as a whole.  (Consider the legal cases of Dred Scott v. Sandford, Brown v. The Board of Education of Topeka, or more recently, The United States v. Arizona over the SB 1070.)
      • Visual Basic (...as well as other programming languages) have the ability for 'sharing' variables that are created at the 'municipal' level with other municipalities... including the 'state' and 'nation' as a whole.

Dim, Private, and Public - are keywords used to declare variables and help define their scope.  The Private and Public keywords can only be used in the General Declarations module.  If a variable is declared with the Private keyword then its scope is only that module.  So, if in the General Declarations module of Worksheet1 you declare: Private item1 As Integer... then the variable item1 is only visible to/accessible by procedures within Worksheet1.  If the same variable was declared: Public item1 As Integer... then it would be visible to/accessible by other worksheets (...and procedures within those worksheets.)

InputBox() - is a function built into Visual Basic (intrinsic function) which prompts users to enter/type in data that will be used by the program.  It is imporant to know that the data entered into the InputBox() text box... is string data... if you want to collect numeric data and then do mathematical calculations with it... you may have to convert the data from string to a numeric data type (Integer, Decimal, etc.) after it is retrieved.
