Event-driven programming - writing programs in which the algorithms are based on user interaction; the user invokes events (clicking, typing text, selecting text, etc.) which causes the program algorithm to perform certain tasks.  (Typically, events are invoked by the 'user' but events can be invoked automatically by program such as a Form's "Load" event or a Timer's "Tick" event.)

Event procedures - self-contained blocks of code that require some type of stimulus.

Comments (remarks) - notes left in the code, by the programmer(s,) to help describe the function of the program.  Comments make it easier to find problems within code, or add different features to the code at a later time.  I feel it is important that you comment your code!
